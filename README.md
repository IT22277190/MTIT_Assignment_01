## Overview

This repository contains two fully functional **Task Management CRUD applications**, each generated independently using a different Generative AI platform:

| Folder | AI Tool Used | Backend | Frontend |
|---|---|---|---|
| `chatGPT/` | ChatGPT-5.3 | FastAPI (Python) | React + Vite + Tailwind CSS |
| `gemini/` | Gemini 3 Pro | FastAPI (Python) | React + Vite + Tailwind CSS |

The purpose of this project is to compare the output quality, code structure, constraint adherence, and reasoning capability of ChatGPT-5.3 vs Gemini 3 Pro when used for real-world software engineering tasks.

## Application Features

Both implementations support the following CRUD operations:

- **Create** — Add a new task with title, description, and completion status
- **Read** — View all tasks displayed in a responsive card grid
- **Update** — Edit an existing task's details or toggle completion status
- **Delete** — Remove a task permanently

### API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/tasks` | Retrieve all tasks |
| `POST` | `/tasks` | Create a new task |
| `PUT` | `/tasks/{task_id}` | Update an existing task by ID |
| `DELETE` | `/tasks/{task_id}` | Delete a task by ID |

### Task JSON Schema

```json
{
  "id": 1,
  "title": "Task title",
  "description": "Task description",
  "is_completed": false
}
```

## Running the Applications

Each project (chatGPT and gemini) has its own backend and frontend. Run them independently.

### Backend (FastAPI)

```bash
# Navigate to the project folder (chatGPT or gemini)
cd chatGPT

# Install Python dependencies
pip install -r requirements.txt

# Start the API server
uvicorn main:app --reload
```

Backend runs at: **http://localhost:8000**
Interactive API docs (Swagger UI): **http://localhost:8000/docs**

---

### Frontend (React + Vite)

```bash
# Navigate to the frontend folder
cd chatGPT/frontend

# Install Node dependencies
npm install

# Start the development server
npm run dev
```

Frontend runs at: **http://localhost:5173**

> Repeat the same steps using the `gemini/` folder to run the Gemini 3 Pro version.

## Manual Improvements Applied

Two engineering improvements were made to the ChatGPT-5.3 output after generation:

1. **Added CORS Middleware** — Required to allow the React frontend (port 5173) to communicate with the FastAPI backend (port 8000) without browser cross-origin blocking.

2. **Fixed `readOnly` checkbox** — Replaced the non-interactive completion checkbox on task cards with a functional `onChange` handler that fires a `PUT` request to toggle task status.

## Debugging Task

An intentional bug was introduced into `chatGPT/main.py` by removing the `raise HTTPException(status_code=404)` from the `update_task` route. This caused the API to return `HTTP 500 Internal Server Error` instead of `HTTP 404 Not Found` when a non-existent task ID was requested — demonstrating how FastAPI's response model validation pipeline fails silently when `None` is returned against a typed `response_model`.

The bug was reproduced and validated using the FastAPI Swagger UI at `http://localhost:8000/docs`.
